# Button spec

## Purpose
Used for primary and secondary user actions.

## Visual variants
- primary
- secondary
- ghost
- danger

## Sizes
- sm
- md
- lg

## Props
- variant?: `'primary' | 'secondary' | 'ghost' | 'danger'`
- size?: `'sm' | 'md' | 'lg'`
- loading?: `boolean`
- leftIcon?: `ReactNode`
- rightIcon?: `ReactNode`
- Inherits native button props via `ButtonHTMLAttributes<HTMLButtonElement>`

## Behavior
- Renders native `button`.
- Applies `ui-button` + variant/size classes.
- Loading state disables interaction and shows `Loading...` label.
- `disabled` is true when `disabled || loading`.

## Accessibility
- Native button semantics are preserved.
- Keyboard accessible by default.
- For icon-only usage, provide an accessible name (for example `aria-label`).

## Styling
- Uses `.ui-button`.
- Variant classes: `.ui-button--primary`, `.ui-button--secondary`, `.ui-button--ghost`, `.ui-button--danger`.
- Size classes: `.ui-button--sm`, `.ui-button--md`, `.ui-button--lg`.
- Colors come from CSS variables.

## Deliverables
- React component
- SCSS style file
- Storybook stories
