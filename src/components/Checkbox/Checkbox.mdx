# Checkbox spec

## Purpose
Provides Ant Design-compatible checkbox behavior for single and grouped selections.

## Props
- `CheckboxProps`
- `indeterminate?: boolean`
- `checked?: boolean`
- `defaultChecked?: boolean`
- `disabled?: boolean`
- `onChange?: (event: CheckboxChangeEvent) => void`
- `rootClassName?: string`
- `classNames?: { root?: string; icon?: string; label?: string }`
- `styles?: { root?: CSSProperties; icon?: CSSProperties; label?: CSSProperties }`
- plus Ant-compatible core props (`prefixCls`, `skipGroup`, `required`, `autoFocus`, `name`, `value`, `tabIndex`, and input attributes)

- `Checkbox.Group` props (`CheckboxGroupProps<T>`)
- `options?: Array<CheckboxOptionType<T> | string | number>`
- `value?: T[]`
- `defaultValue?: T[]`
- `onChange?: (checkedValues: T[]) => void`
- `name?: string`
- `disabled?: boolean`

## Behavior
- Uses native `<input type="checkbox">` and supports controlled/uncontrolled usage.
- Supports `indeterminate` state with mixed aria state (`aria-checked="mixed"`).
- Supports Ant-compatible group composition via `Checkbox.Group` and option arrays.
- Checkbox inside a group syncs checked state by matching `value` against group values.

## Accessibility
- Native checkbox semantics with keyboard interaction and visible focus style.
- Label click toggles checkbox through associated native control.
- Group renders semantic container (`role="group"`).

## Styling
- Uses `.ui-checkbox*` classes in `Checkbox.ant.scss` for Ant-like visuals.
- Checked and indeterminate states use theme primary color.
- Dark-mode hover/active/focus borders stay neutral (`--ui-control-border-*`, `--ui-focus`) for control contrast.

## Deliverables
- React component
- SCSS style file
- Storybook stories
