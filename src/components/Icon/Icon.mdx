# Icon spec

## Purpose
Render reusable SVG icons from an internal icon map with per-icon `viewBox` support.

## Available icons
- Raw icon keys from all parsed sources are supported:
  - Ant Design: 789
  - Bootstrap: 1,953
  - Font Awesome: 2,020
  - Material: 10,605
  - VT: 1,148
- Legacy aliases are still supported for backwards compatibility (for example: `search`, `close`, `warningCircle`).
- Use Storybook `Components/Icon` family pages (`AntDesign`, `Bootstrap`, `FontAwesome`, `Material`, `VT`) to browse the full icon catalog.

## Props
- name: `IconName` (required, accepts raw source key or legacy alias)
- size?: `number | string`
- Inherits native SVG props via `SVGProps<SVGSVGElement>`

## Behavior
- Renders `<svg>` with `fill="currentColor"`.
- Uses icon-specific `viewBox` values to keep each glyph fit to its own SVG bounds.
- Uses `size` for both width and height.
- Default size is `1em` for CSS-friendly alignment and scaling.
- Resolution order:
  1. Exact raw key lookup (for example: `mat_accessAlarmRound`)
  2. Legacy alias lookup (for example: `search`, `close`)
  3. Fallback icon for unknown names

## Accessibility
- Sets `aria-hidden="true"` by default.
- For semantic icons, consumers should pass proper ARIA props.

## Styling
- Uses `.ui-icon` class and consumer-provided `className`.
- Defaults to `width`/`height` of `1em` and inline alignment via component styles.
- Color inherits from surrounding text unless overridden.

## Deliverables
- React component
- Internal icon definition map (`viewBox` + path list)
- Story usage across component demos
