# Table spec

## Purpose
Theme-aware data table component rendered with native table markup (`<table>`, `<thead>`, `<tbody>`) and library-owned styling.

## API
- `columns`: Column definitions (`TableColumnType<RecordType>`) for title, data access, renderers, sorting, and filters.
- `dataSource`: Row records.
- `pagination`: Optional pagination config (`TablePaginationConfig`) with controlled/uncontrolled page + page-size.
- `loading`, `bordered`, `size`, `locale.emptyText`, and `rowKey`.
- Expand controls: `expandedRowRender`, `rowExpandable`, `expandedRowKeys`, `defaultExpandedRowKeys`, `onExpand`, `onExpandedRowsChange`.
- Events: `onChange`, `onRow`, `onHeaderRow`, `onScroll`.

## Exported types
- Core: `TableProps`, `TableColumnType`, `ColumnsType`, `AnyObject`.
- Paging/locale: `TablePaginationConfig`, `TablePaginationPosition`, `TableLocale`.
- Filtering/sorting: `ColumnFilterItem`, `CompareFn`, `SortOrder`, `FilterValue`.
- Event aliases: `TableOnChange`, `TableOnRow`, `TableOnHeaderRow`, `TableOnScroll`, `TableOnExpand`, `TableOnExpandedRowsChange`, `TableOnChangeExtra`.

## Styling
- Root class: `.ui-table`.
- Uses token-based CSS variables for border, surface, hover, focus, and primary interactive states.
- No runtime styling libraries and no dependency on external table component markup.

## Composition
- Built with library primitives for interactions: `Checkbox` for column filter options, `Button` for row expansion + pagination nav controls, and `Select` for page-size selection.
- Cell renderers can compose other UI components such as `Tag`, `Button`, `Avatar`, etc.
