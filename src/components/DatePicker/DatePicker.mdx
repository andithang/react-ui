# DatePicker spec

## Purpose
Ant Design-style date input with trigger field, popup calendar panel, navigation, and clear action.

## Props
- `label?: string`
- `hint?: string`
- `error?: string`
- `picker?: 'date' | 'month' | 'year'` (default: `date`)
- `size?: 'small' | 'middle' | 'large'` (default: `middle`)
- `status?: 'error' | 'warning'`
- `allowClear?: boolean` (default: `true`)
- `disabled?: boolean`
- `placeholder?: string` (default: `Select date`)
- `format?: string` (default: `YYYY-MM-DD`)
- `value?: Date | null`
- `defaultValue?: Date`
- `open?: boolean`
- `defaultOpen?: boolean`
- `onOpenChange?: (open: boolean) => void`
- `onChange?: (date: Date | null, dateString: string) => void`
- `disabledDate?: (date: Date) => boolean`
- `id?: string`
- `className?: string`

## Behavior
- Uses a button-like control to open a popup panel.
- Supports calendar day grid, month picker, and year picker.
- Emits both raw `Date` and formatted date string in `onChange`.

## Accessibility
- Trigger is keyboard-focusable button with `aria-haspopup="dialog"`.
- Panel closes on outside click.

## Styling
- Uses `ui-date-picker` namespace and token-based styling.
